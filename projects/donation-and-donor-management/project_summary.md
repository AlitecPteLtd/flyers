# Project Summary — Donation and Donor Management

## Business Context

This flyer is positioned generically as Donation and Donor Management instead of using the source project name.

The project focuses on donation and donor contact records, legacy database continuity, personal information fields, partner grade descriptions, and Odoo Website homepage migration.

## Source Modules Reviewed

- `custom_contact`: adds donor-style details and contact fields to `res.partner`.
- `website_migration`: migrates homepage content into Odoo Website.

## Verified Customization Points

- Donor number and old database number.
- NRIC number and house/unit number.
- First-name and last-name fields with fallback full-name creation.
- Last payment/donation date, old last update date, and expiry date.
- Partner grade description field.
- Contact list and form view additions for faster donor review.
- Dependencies on OCA personal information, birthdate, nationality, deduplication, and partner contact modules.
- Website homepage with hero, programs, events snippet, statistics, and centre story.

## Positioning

- **Slug:** `donation-and-donor-management`
- **Title:** Donation and Donor Management
- **Subtitle:** Donation Records, Donor Profiles & Website Migration with Odoo
- **Message:** trusted donor data, legacy continuity, and website migration.

## Avoid Claiming

- Ceremony, puja, offering, memorial tablet, or POS workflow as the main scope.
- Do not use the source project/client name in the flyer title or folder path.
